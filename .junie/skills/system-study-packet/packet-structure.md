# 8-Principle packet blueprint (condensed)

Build sections **in this order**. Adjust depth by learner level (see end).

## Front matter

- Title + subtitle ("A Structured Study Packet")
- How to use (Step 1 Understanding, Step 2 Automaticity)
- Table of 8 principles
- TOC (linked list in Markdown)

## Step 1 — Understanding

### Principle 1 — Map of the system

- Flow table (stages in order + one line each + main code paths)
- Components/subsystems table
- **Map takeaway** (1–2 sentences)

### Principle 2 — Clear explanations

- 5–9 subsections as questions ("What is X?", "How does Y work?")
- One paragraph each; define terms; use real file/class names
- **Explanation takeaway** (core idea + common misconception)

### Principle 3 — Different media

Include at least three:

- One-line summary
- Text diagram (flow or layers)
- Analogy
- Comparison table (vs common confusion)

- **Media takeaway**

### Principle 4 — Short lessons

- 4–6 numbered micro-lessons (2 min each)
- **Short-lessons takeaway**

## Step 2 — Automaticity

Intro: understanding ≠ remembering.

### Principle 5 — Test yourself

- Quiz 8–12 questions
- Answer key (model answers)
- Flashcards 10–15 (front | back table)
- Only test what was taught

### Principle 6 — Wait to review

- Table: Today, Day 1, 3, 7, 14, 30 — activity + checkbox
- One sentence on why spacing works

### Principle 7 — Mix it up

- Interleaved quiz 8–12 (jump topics)
- Separate answer key
- One sentence on interleaving

### Principle 8 — Don't stop

- Stages table (first correct → comfortable → automatic)
- Overlearning bullet plan
- **Final takeaway**

## Appendix

- Glossary table (term → one line)
- Links to decomposition doc + specs/handoff

## Sizing

| Level | Ideas in P2 | Lessons | Quiz | Flashcards |
|-------|-------------|---------|------|------------|
| Beginner | ~5 | 4 | 8 | 10 |
| Student | ~7 | 5 | 10 | 12 |
| Professional | ~9 | 6 | 12 | 15 |

Default: **student**.
