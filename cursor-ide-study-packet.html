Step 1 — Understanding

Goal: a correct, simple mental model — what Cursor is made of and how a request flows through it. Get this right and every feature becomes predictable.

Principle 1A map of the system

Cursor is an AI-native code editor — a fork of VS Code by Anysphere that keeps VS Code's extensions, themes, and keybindings, but rebuilds the editing loop around AI. The map has three layers: interaction primitives (how you talk to it), the context engine (what it knows about your code), and the agent runtime (what it can do on its own).

How a request flows

#StageWhat happens
1IntentYou trigger a primitive: Tab, Inline Edit (Ctrl/Cmd+K), Chat (Ctrl/Cmd+L), or Agent/Composer.
2Context assemblyCursor gathers context: open files, @-mentions, the semantic codebase index, and any matching rules.
3Rules + model routingActive rules are prepended; the request is routed to a model (Composer, Claude, GPT-5, o-series), optionally in Max Mode.
4GenerationThe model proposes edits, an answer, or a multi-step plan.
5Action (agent only)In Agent mode it runs terminal commands, reads output, edits files, browses @web, calls MCP tools — looping until done.
6Review & applyYou see a diff, accept/reject per-hunk, and the loop ends or continues.

The components

Map takeaway

Everything in Cursor is one of three things: a primitive you trigger, the context it assembles, or the agent loop it runs. Master those three and the product stops surprising you.

Principle 2Clear, simple explanations

The nine ideas that actually matter. Each is one question, one tight answer.

What makes Cursor different from "VS Code + Copilot"?

Copilot bolts AI onto an editor as a suggestion engine. Cursor rebuilds the editor around an agent: a unified context engine indexes your whole repo, and an agent runtime can run commands and edit many files in a loop. The difference is architectural, not cosmetic — Cursor treats the editor as a runtime the agent operates, not a textbox it autocompletes.

What is Tab, really?

Tab is Cursor's predictive editing model. Beyond completing the current line, it predicts the next edit — often across multiple lines or jumping to another location implied by your change (e.g. updating a call site after you rename a parameter). It is the single most-used feature; the recent Composer-era Tab reached very high acceptance rates (~72%), which is why it feels like the editor is reading your intent.

Inline Edit vs Chat vs Agent — when does each fire?

Inline Edit (Ctrl/Cmd+K) is for a surgical, in-place change to a selection. Chat/Ask (Ctrl/Cmd+L) is for questions and exploration — it reads and explains but doesn't take actions. Agent/Composer is for "go do it": multi-file work where the model plans, runs tools, and applies changes. Rule of thumb: Ask to understand, Inline to tweak, Agent to build.

How does Cursor know about my codebase?

It builds a semantic index: your files are chunked and embedded into vectors so the model can retrieve code by meaning, not just by filename. You steer it explicitly with @-symbols — @file, @folder, @code, @docs, @git, @web. Good context hygiene (point at the right things, don't dump the world) is the biggest lever on output quality.

What is the context window, and what is Max Mode?

The context window is how much text the model can hold at once — by default around 200k tokens (~15k lines). Max Mode expands that ceiling for large models so the agent can hold far more of a big codebase in one pass, at higher token cost. Use it for sprawling refactors; skip it for small, well-scoped edits where extra context just adds noise and spend.

What are Rules and why do they matter more than prompts?

Rules are persistent instructions injected into the model's context every relevant request, so you don't re-type conventions. Modern Cursor uses .cursor/rules/ — multiple .mdc files that load conditionally: Always (every request), Auto-Attached (when files matching a glob are in context), Agent-Requested (the model pulls it when relevant), or Manual (you @-mention it). The legacy single .cursorrules file still works, but folder rules keep the system prompt small as the rule set grows. Rules beat one-off prompts because they're versioned, shared, and automatic.

What is MCP, and how do hooks and skills extend it?

MCP (Model Context Protocol) is an open standard for connecting the agent to external tools and data sources (databases, issue trackers, browsers, internal APIs) through a uniform interface. Hooks run your own logic at lifecycle points (e.g. before/after an edit or command) to gate or shape agent behaviour. Skills are reusable, pinnable capability bundles. Together they turn Cursor from an editor into a programmable agent platform — available on Pro and above.

What are Cloud / Background Agents?

A Cloud Agent runs in an isolated cloud VM with its own terminal, browser, and file system. It can work across multiple repositories in parallel and report results back to your IDE asynchronously — you fire off a task and keep working. "Build in Parallel" spawns subagents to tackle pieces concurrently. This is the shift from "AI in my editor" to "a fleet of agents I dispatch."

How does Cursor cost money — requests, credits, and seats?

Cursor meters AI usage. Tiers: Hobby (free; limited Tab + slow requests), Pro (~$20/mo; unlimited Tab + a pool of fast requests, then a slow queue), Business (~$40/seat/mo; adds SSO, admin controls, and zero-data-retention), Enterprise (custom). "Fast" requests are prioritised compute; heavy operations (Max Mode, big agent runs) consume more. The mental model: Tab is cheap and flat; agentic runs and Max Mode are where spend concentrates.

Explanation takeaway

The one idea that organises all nine: context quality drives everything. Cursor's primitives, index, rules, and Max Mode all exist to put the right code and conventions in front of the model.

Most common misconception: that a bigger model or Max Mode fixes bad output. Usually the fix is better-scoped context and a good rule — not more tokens.

Principle 3Use different media

The same ideas, met in different forms so they stick.

One-line summary

Cursor is a VS Code–based editor rebuilt around an AI agent: a semantic index and @-context feed models that autocomplete (Tab), edit in place, and — in Agent mode — run tools to make multi-file changes, all steered by persistent rules and extended by MCP, hooks, skills, and cloud agents.

Diagram — the request loop

YOU │ trigger ▼ PRIMITIVE ── Tab · Inline(⌘K) · Chat(⌘L) · Agent │ ▼ CONTEXT ENGINE ├─ @-symbols (file/folder/docs/web/git) ├─ semantic codebase index (embeddings) └─ RULES (.cursor/rules/ → Always | Auto | Requested | Manual) │ ▼ MODEL ROUTING ── Composer · Claude · GPT-5 · o-series [+ Max Mode = bigger window] │ ▼ AGENT LOOP (agent only) run cmd → read output → edit files → @web → MCP tool ──┐ ▲___________________ repeat until done ___________│ │ ▼ DIFF → you accept/reject per hunk (or dispatch to a CLOUD AGENT: isolated VM, parallel, async)

Analogy

Think of Cursor as a workshop with a skilled apprentice. Tab is the apprentice finishing your sentence as you work. Inline Edit is handing them one part with "make this fit." Chat is asking them about the project without letting them touch anything. Agent is saying "build this shelf" and letting them fetch tools, cut wood, and check their work — while you inspect the result. Rules are the shop's posted standards the apprentice always follows. MCP is giving them keys to the supply room next door. Cloud agents are hiring a second apprentice who works in another room and reports back.

Principle 4Short lessons

Six micro-lessons. Each is a couple of minutes. Repetition with earlier sections is intentional.

Lesson 1 — The four primitives, and choosing between them

Tab predicts edits as you type; Inline Edit (⌘K) changes a selection in place; Chat (⌘L) explains without acting; Agent/Composer plans and executes multi-file work with tools. Pick by blast radius: bigger and more autonomous → move rightward toward Agent.

Lesson 2 — Context is something you steer

The semantic index gives the model a searchable memory of your repo, but you get the best results by pointing explicitly with @file, @folder, @docs, @git, @web. Less, but relevant, beats dumping everything. 🛠 Solo senior dev

Lesson 3 — Rules are your standards-as-code

Put conventions in .cursor/rules/ as scoped .mdc files: Always for universal style, Auto-Attached globs for per-area rules (e.g. only when editing *.test.ts), Agent-Requested for situational guidance. This is how a team gets consistent agent output without everyone re-prompting. 👥 Manager

Lesson 4 — Models and Max Mode are a cost/benefit dial

Cursor routes among Composer, Claude, GPT-5, and o-series models; Max Mode trades money for a much larger context window. Reach for Max Mode on large, cross-cutting refactors; otherwise the default window is faster, cheaper, and often cleaner. 🏢 CEO

Lesson 5 — Extend the agent with MCP, hooks, and skills

MCP connects external tools/data behind one protocol; hooks run your logic at agent lifecycle points (gate a command, lint after an edit); skills bundle reusable capabilities you can pin. This is how Cursor becomes a platform tailored to your stack. ♾ Unlimited

Lesson 6 — Cloud agents change the unit of work

Background/Cloud Agents run in isolated VMs, in parallel, across repos, reporting back asynchronously. The skill shifts from "writing code with help" to "dispatching and reviewing agent work" — decomposition and review become the bottleneck, not typing. ♾ Unlimited 👥 Manager

Short-lessons takeaway

Choose primitives by blast radius, steer context deliberately, encode standards as rules, treat Max Mode as a cost dial, extend via MCP/hooks/skills, and learn to dispatch and review cloud agents.

Step 2 — Automaticity

Understanding fades. These four practices — retrieval, spacing, mixing, overlearning — turn what you just read into fast, durable recall. This is the half most people skip; it's the half that works.

Principle 7Mix it up

This quiz jumps deliberately between primitives, context, rules, cost, security, and extension — out of order — so you practise choosing the right idea, not reciting a sequence.

B1. You're a CEO weighing rollout: which tier gives zero-data-retention and SSO, and why care?
Business (~$40/seat). ZDR + SSO matter for compliance and IP protection when code can't leave under retention.
B2. A teammate's agent keeps ignoring your naming convention. Fix it at the team level — how?
Add an Always (or glob-scoped Auto-Attached) rule in .cursor/rules/, commit it. Now every agent run inherits it. 👥 Manager
B3. You only want to rename a variable in a 12-line function. Which primitive, and why not Agent?
Inline Edit (⌘K) or just Tab — minimal blast radius. Agent is overkill and adds review cost.
B4. The agent gives wrong output on a big refactor. Name two fixes before reaching for Max Mode.
Tighten context scope (point at the right files via @) and add/clarify a rule. Context > raw tokens.
B5. What's the architectural difference between Cursor and "VS Code + Copilot"?
Cursor is built around an agent (unified context engine + agent runtime that runs tools); Copilot bolts suggestions onto the editor.
B6. You need the agent to query your internal Postgres and your Jira. What capability connects those?
MCP servers expose those systems to the agent through one protocol. ♾ Unlimited
B7. Why might you dispatch a Cloud Agent instead of running the agent in your editor?
To run long/parallel work across repos asynchronously in isolated VMs while you keep working; review results later.
B8. Distinguish Chat/Ask from Agent in one sentence.
Ask reads and explains without acting; Agent plans and executes changes with tools.
B9. What's the cost intuition: where does spend concentrate across Cursor usage?
Tab is cheap/flat; agentic runs and Max Mode consume the most (fast requests / larger context). 🏢 CEO
B10. You want a rule that only applies when editing test files. Which activation type and mechanism?
Auto-Attached with a glob (e.g. *.test.ts) in a .cursor/rules/ .mdc file.
B11. Name a hook use-case that improves safety on an autonomous agent.
Gate/validate a terminal command before it runs, or auto-lint/format after an edit — logic at agent lifecycle points. ♾ Unlimited
B12. As a solo senior dev, what daily habit most improves Cursor's results?
Context hygiene — explicitly @-mention the relevant files/docs and keep rules current, rather than dumping the whole repo. 🛠 Solo senior dev

Why mixing helps: real work never arrives labelled by topic. Interleaving trains the retrieval cue "which idea fits this situation," which is the skill you actually use.

Principle 8Don't stop

Most people quit the moment they first answer right. But the practice after first success is what makes recall fast and automatic under pressure. Push past "I got it once."

Overlearning plan

  • Run the flashcards until you get three flawless rounds in a row.
  • Once a month, do a cold self-test of both quizzes — no notes.
  • Teach it: explain MCP, the rule activation types, and the Max-Mode trade-off to a colleague. Teaching exposes the gaps.
  • Apply it: on your next real task, deliberately pick the primitive by blast radius and write one new .cursor/rules/ file. Practice in production beats re-reading.

Final takeaway

Step 1 gave you the model — primitive → context → model → agent loop → diff, extended by rules, MCP, and cloud agents. Step 2 makes it permanent: test, space, mix, and overlearn. Knowledge becomes durable through practice after first success — so don't stop at "I understood it."
