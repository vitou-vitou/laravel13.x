# UI adoption workflow (read later)

Three formats, same substance — pick by how you want to use it.

| File | Format | Best for |
|------|--------|----------|
| [8-principle-study-ui-adoption.md](./8-principle-study-ui-adoption.md) | 8-principle study | Learning, quizzes, spaced review |
| [PRODUCT.md](./PRODUCT.md) + [DESIGN.md](./DESIGN.md) | Impeccable context | AI prompts and project constraints |
| [PRO-DEV-RUNBOOK.md](./PRO-DEV-RUNBOOK.md) | Pro dev runbook | Checklists and PR execution |

## Start here

1. **First time:** Read runbook [Phase 0 — Intake](./PRO-DEV-RUNBOOK.md#phase-0--intake-mandatory-30-min).
2. **Before any UI AI session:** Copy `DESIGN.md` guardrails into your example app (or symlink).
3. **To remember habits:** Work through the [8-principle packet](./8-principle-study-ui-adoption.md) once, then use Principle 6 schedule.
