## How to use this document

**Grammarly workflow**
1. Paste each day's **Body** into Grammarly (browser extension or desktop).
2. Set goals: *Professional*, *Confident*, *Warm* — avoid *Persuasive/Excited* (keeps hustle-bro out).
3. Fix only clarity and tone; keep product nouns (`publish log`, `pending_approval`, `weekly metrics`) unchanged.
4. Save approved versions in your ESP (Loops, Resend, Mailchimp) or social scheduler as drafts.

**Superhuman-style email rules**
- Subject ≤ 45 characters when possible; front-load the verb.
- First line = the point. No "Hope you're well."
- One CTA per message. Link text describes the action ("Review this week's queue").
- Use `---` mental break before CTA in longer emails.
- Send operator-facing mail from a human name (`Alex — Creator Operator`), not `noreply@`.

**Scheduling tips**
- **Relaxing time** entries (8:00–9:30 PM local): wind-down tone; good for operators who batch on Sunday night.
- **Morning** entries (7:30–9:00 AM): SMS and short LinkedIn — catch inbox before the batch day.
- **Midday** (12:00–1:00 PM): creator-facing approvals; they're between content blocks.
- Tag `Multi` = same core message adapted for email + one social line in the body.

**Rotation**
- Months 1–2: spreadsheet pain → weekly batch habit.
- Months 3–4: approvals, settlement trust, creator persona.
- Months 5–6: TikTok import, n8n webhooks, billing, permission to rest.
