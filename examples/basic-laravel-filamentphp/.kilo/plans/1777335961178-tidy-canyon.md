# Todo CRUD API Specification - Laravel 13

## Project Context
- Laravel 13.x application with minimal existing code
- Current routes: web.php with signed route demo
- Need: Production-grade REST API for Todo resource

## API Design Principles (2026 Best Practices)
- Resource-oriented REST design (nouns, plural)
- JSON:API response format with consistent envelope
- API versioning via URL prefix (`/api/v1/`)
- Proper HTTP methods and status codes
- FormRequest validation for all inputs
- API Resources for response formatting
- Rate limiting for sensitive endpoints
- Paginated collection endpoints

## Todo Resource Specification

### Data Model
```json
{
  "id": "integer",
  "title": "string (required, max:255)",
  "description": "string|null",
  "status": "enum:[pending,in_progress,completed] (default:pending)",
  "priority": "enum:[low,medium,high] (default:medium)",
  "due_date": "date|null",
  "completed_at": "timestamp|null",
  "created_at": "timestamp",
  "updated_at": "timestamp"
}
```

### Endpoints

#### 1. List Todos (Paginated)
```
GET /api/v1/todos
```
Query Parameters:
- `page` (int) - Page number (default: 1)
- `per_page` (int) - Items per page (default: 15, max: 100)
- `status` (string) - Filter by status (pending|in_progress|completed)
- `priority` (string) - Filter by priority (low|medium|high)
- `search` (string) - Search in title/description
- `sort` (string) - Sort field (default: created_at)
- `direction` (string) - Sort direction (asc|desc, default: desc)

Response: 200 OK
```json
{
  "data": [
    {
      "id": 1,
      "title": "Complete API spec",
      "description": "Design REST endpoints",
      "status": "in_progress",
      "priority": "high",
      "due_date": "2026-04-30",
      "completed_at": null,
      "created_at": "2026-04-28T07:00:00Z",
      "updated_at": "2026-04-28T07:30:00Z"
    }
  ],
  "meta": {
    "current_page": 1,
    "per_page": 15,
    "total": 1,
    "last_page": 1
  }
}
```

#### 2. Create Todo
```
POST /api/v1/todos
```
Request Body:
```json
{
  "title": "string (required, max:255)",
  "description": "string|null",
  "status": "enum:[pending,in_progress,completed] (default:pending)",
  "priority": "enum:[low,medium,high] (default:medium)",
  "due_date": "date|null"
}
```

Response: 201 Created
```json
{
  "data": {
    "id": 1,
    "title": "Complete API spec",
    "description": "Design REST endpoints",
    "status": "pending",
    "priority": "high",
    "due_date": "2026-04-30",
    "completed_at": null,
    "created_at": "2026-04-28T07:35:00Z",
    "updated_at": "2026-04-28T07:35:00Z"
  }
}
```

Validation Errors: 422 Unprocessable Content
```json
{
  "message": "The given data was invalid.",
  "errors": {
    "title": ["The title field is required."]
  }
}
```

#### 3. Get Single Todo
```
GET /api/v1/todos/{id}
```

Response: 200 OK
```json
{
  "data": {
    "id": 1,
    "title": "Complete API spec",
    "description": "Design REST endpoints",
    "status": "pending",
    "priority": "high",
    "due_date": "2026-04-30",
    "completed_at": null,
    "created_at": "2026-04-28T07:00:00Z",
    "updated_at": "2026-04-28T07:30:00Z"
  }
}
```

Not Found: 404 Not Found
```json
{
  "message": "Todo not found."
}
```

#### 4. Update Todo
```
PATCH /api/v1/todos/{id}
```
Request Body (all fields optional):
```json
{
  "title": "string (max:255)",
  "description": "string|null",
  "status": "enum:[pending,in_progress,completed]",
  "priority": "enum:[low,medium,high]",
  "due_date": "date|null"
}
```

Response: 200 OK
```json
{
  "data": {
    "id": 1,
    "title": "Updated title",
    "description": "Updated description",
    "status": "in_progress",
    "priority": "medium",
    "due_date": "2026-05-01",
    "completed_at": null,
    "created_at": "2026-04-28T07:00:00Z",
    "updated_at": "2026-04-28T08:00:00Z"
  }
}
```

#### 5. Delete Todo
```
DELETE /api/v1/todos/{id}
```

Response: 204 No Content

Not Found: 404 Not Found

#### 6. Mark Todo as Complete
```
PATCH /api/v1/todos/{id}/complete
```
Sets `status = 'completed'` and `completed_at = now()`

Response: 200 OK (returns updated todo)

#### 7. Mark Todo as Incomplete
```
PATCH /api/v1/todos/{id}/incomplete
```
Sets `status = 'pending'` and `completed_at = null`

Response: 200 OK (returns updated todo)

## Implementation Steps

1. Create Todo model and migration
2. Create FormRequest classes (StoreTodoRequest, UpdateTodoRequest)
3. Create API Resource (TodoResource)
4. Create API Resource Collection (TodoCollection)
5. Create TodoController with 7 actions
6. Define API routes in routes/api.php
7. Add rate limiting for write operations
8. Add API versioning prefix
9. Write tests for all endpoints

## Technical Details

- Use Laravel API Resources for consistent JSON responses
- Use FormRequest classes for validation logic
- Use Eloquent query scopes for filtering
- Implement pagination via Laravel's built-in paginate()
- Use Route model binding for resource retrieval
- Apply throttle middleware for rate limiting
- Follow PSR-12 coding standards
